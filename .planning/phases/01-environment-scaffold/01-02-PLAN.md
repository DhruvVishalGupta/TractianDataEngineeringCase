# Plan 01-02: Requirements File and Gitignore

## Objective

Produce a `requirements.txt` that exactly records every dependency already
installed in the environment, and a `.gitignore` that prevents raw data,
outputs, logs, secrets, and cache files from entering version control.

## Tasks

1. **Write `requirements.txt` with pinned versions.**

   Run `pip freeze` to get the current environment snapshot, then write
   `requirements.txt` to the project root. The file must include at minimum:

   ```
   requests
   pydantic>=2.0
   openpyxl
   pandas
   tqdm
   httpx
   tenacity
   fastapi
   uvicorn[standard]
   geopy
   rapidfuzz
   ```

   Use `pip freeze` output to pin exact versions for reproducibility:

   ```bash
   cd /c/Users/dhruv/TractianCase-DhruvGupta
   pip freeze > requirements.txt
   ```

   After writing, open `requirements.txt` and verify these package names are
   present (pip may have installed them under slightly different names — check
   and correct if needed):
   - `requests` (not `urllib3` alone)
   - `pydantic` (v2.x preferred; if v1.x is installed, note it in a comment
     but do not upgrade automatically — that would break schema.py)
   - `rapidfuzz` (used by deduplicator.py)
   - `geopy` (used by validator.py for geocoding)
   - `fastapi` and `uvicorn` (for the API layer in Phase 3)
   - `pandas` and `openpyxl` (for CSV/XLSX output)

   If any of these are absent from `pip freeze` output, install them and
   re-run freeze:
   ```bash
   pip install <missing-package>
   pip freeze > requirements.txt
   ```

2. **Create `.gitignore`.**

   Write the following content to `.gitignore` in the project root. Do not
   overwrite an existing `.gitignore` — if one exists, append only the
   sections that are missing.

   ```
   # Pipeline data — can be regenerated; do not commit raw crawl results
   data/raw/
   data/processed/

   # Pipeline outputs — final deliverables go here, excluded from git
   outputs/

   # Logs
   logs/

   # Environment secrets
   .env
   .env.*
   !.env.example

   # Python cache
   __pycache__/
   *.pyc
   *.pyo
   *.pyd
   .Python
   *.egg-info/
   dist/
   build/
   .eggs/

   # Virtual environments
   venv/
   .venv/
   env/
   .env/

   # IDE
   .vscode/
   .idea/
   *.swp
   *.swo

   # OS
   .DS_Store
   Thumbs.db

   # Jupyter
   .ipynb_checkpoints/
   *.ipynb
   ```

   After writing, run `git status` and confirm that `data/raw/`, `outputs/`,
   and `logs/` directories no longer appear as untracked paths in the output.
