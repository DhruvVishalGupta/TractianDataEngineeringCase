# Plan 01-01: Module Import Verification and Fix

## Objective

Run each pipeline module through Python's import machinery and resolve any
NameError, ImportError, circular-import, or missing-dependency failures that
prevent a clean import. No functional changes — only structural/import fixes.

## Tasks

1. **Run import smoke test for every pipeline module.**

   Execute the following commands one at a time from the project root.
   Capture stdout+stderr for each. Record which modules fail and the exact
   error message.

   ```bash
   cd /c/Users/dhruv/TractianCase-DhruvGupta

   python -c "from src.pipeline import config; print('config OK')"
   python -c "from src.pipeline import logger; print('logger OK')"
   python -c "from src.pipeline import schema; print('schema OK')"
   python -c "from src.pipeline import raw_store; print('raw_store OK')"
   python -c "from src.pipeline import icp_scorer; print('icp_scorer OK')"
   python -c "from src.pipeline import ollama_client; print('ollama_client OK')"
   python -c "from src.pipeline import scraper; print('scraper OK')"
   python -c "from src.pipeline import searcher; print('searcher OK')"
   python -c "from src.pipeline import edgar; print('edgar OK')"
   python -c "from src.pipeline import firmographics; print('firmographics OK')"
   python -c "from src.pipeline import classifier; print('classifier OK')"
   python -c "from src.pipeline import validator; print('validator OK')"
   python -c "from src.pipeline import deduplicator; print('deduplicator OK')"
   python -c "from src.pipeline import output_csv; print('output_csv OK')"
   ```

   For any module that does NOT print "OK", read the file, identify the root
   cause, and fix it in place. Common failure modes to look for:
   - Missing import at top of file (add the missing `import` statement)
   - Relative import used outside a package context (ensure `src/__init__.py`
     exists — create it as empty if missing)
   - A dependency installed under a different package name than expected
     (check `pip show <name>` and adjust the import accordingly)
   - `from __future__ import annotations` missing when using `list[...]` or
     `Optional[...]` type hints in Python 3.9 (add it as the first line)

   Do NOT change any business logic. Fix only the import-layer problem.

2. **Run a combined all-modules import to confirm no circular dependencies.**

   After all individual imports pass, run:

   ```bash
   python -c "
   from src.pipeline import (
       config, logger, schema, raw_store,
       icp_scorer, ollama_client, scraper,
       searcher, edgar, firmographics,
       classifier, validator, deduplicator,
       output_csv,
   )
   print('ALL MODULES OK')
   "
   ```

   If a circular import surfaces (e.g., `ImportError: cannot import name X`),
   resolve it by moving the offending import inside the function body that uses
   it rather than at module-level. This is the minimal-impact fix.

   Also verify `src/__init__.py` exists (create empty if not):
   ```bash
   touch src/__init__.py
   ```
